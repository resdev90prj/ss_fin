# Codex Context Template

## Visao geral do projeto
- Descreva em 2-4 linhas o que o projeto e hoje.
- Indique o estagio atual (`MVP`, `producao`, `refatoracao`, etc.).
- Informe data da ultima atualizacao deste contexto.

## Objetivo do sistema
- Explique o problema de negocio que o sistema resolve.
- Liste os principais resultados esperados para o usuario final.

## Stack tecnologica
- Liste linguagens, framework(s), banco de dados e bibliotecas principais.
- Informe versoes relevantes quando conhecidas.
- Marque como `a validar` quando a versao nao for confirmada.

## Arquitetura
- Descreva o padrao arquitetural adotado (ex.: MVC, camadas, monolito).
- Explique o fluxo principal de requisicao (entrada, controller, model, view).
- Registre pontos de acoplamento importantes.

## Estrutura de diretorios
- Liste as pastas principais e o papel de cada uma.
- Inclua caminhos criticos para bootstrap, config, rotas e UI.

## Modulos principais
- Liste modulos implementados e estado (`completo`, `parcial`, `a validar`).
- Cite controllers/models/views mais relevantes por modulo.

## Regras de negocio
- Registre regras criticas que impactam comportamento funcional.
- Inclua validacoes, bloqueios, permissoes e regras de calculo.

## Convencoes tecnicas
- Padroes de codigo e organizacao usados no projeto.
- Convencoes de nomenclatura, roteamento, seguranca e tratamento de erros.

## Decisoes arquiteturais
- Liste decisoes ja tomadas e o motivo (quando conhecido).
- Registre trade-offs e impactos para evolucao futura.

## Integracoes externas
- Liste APIs, servicos externos, webhooks, cron, filas, gateways, etc.
- Informe quais integracoes sao obrigatorias e quais sao opcionais.

## Pontos de atencao
- Registre riscos tecnicos, dividas tecnicas e fragilidades conhecidas.
- Inclua limitacoes de ambiente e dependencias sensiveis.

## Pendencias
- Liste pendencias abertas com prioridade (`alta`, `media`, `baixa`).
- Marque claramente o que depende de validacao externa.

## Proximos passos
- Defina 3-7 proximos passos acionaveis.
- Preferir tarefas pequenas, verificaveis e com impacto claro.
